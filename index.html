Model 3 — GCN (graph convolutional network)

Walk through the propagation rule and the code.

from torch_geometric.nn import GCNConv class GCN(nn.Module): def __init__(self, in_dim, hidden_dim, out_dim): super().__init__() self.conv1 = GCNConv(in_dim, hidden_dim) self.conv2 = GCNConv(hidden_dim, out_dim) def forward(self, x, edge_index): x = F.relu(self.conv1(x, edge_index)) # aggregate neighbors x = F.dropout(x, 0.5, training=self.training) return self.conv2(x, edge_index)
"Now edge_index is passed to forward(). GCNConv applies the formula: H⁽ˡ⁺¹⁾ = σ(D̂⁻½ Â D̂⁻½ H⁽ˡ⁾ W⁽ˡ⁾) — it averages each node's features with its neighbors', weighted by degree."

Model 4 — GraphSAGE

Explain inductive learning and the CONCAT aggregation.

from torch_geometric.nn import SAGEConv class GraphSAGE(nn.Module): def __init__(self, in_dim, hidden_dim, out_dim): super().__init__() self.conv1 = SAGEConv(in_dim, hidden_dim) self.conv2 = SAGEConv(hidden_dim, out_dim) def forward(self, x, edge_index): x = F.relu(self.conv1(x, edge_index)) x = F.dropout(x, 0.5, training=self.training) return self.conv2(x, edge_index)
"GraphSAGE samples a fixed-size neighborhood and concatenates the node's own embedding with the aggregated neighbor embedding — unlike GCN which averages them. This makes it more inductive: it can generalize to unseen nodes."

Model 5 — GAT (graph attention network)

The most complex model — explain the attention mechanism.

from torch_geometric.nn import GATConv class GAT(nn.Module): def __init__(self, in_dim, hidden_dim, out_dim, heads=8): super().__init__() self.conv1 = GATConv(in_dim, hidden_dim, heads=heads, dropout=0.6) self.conv2 = GATConv(hidden_dim * heads, out_dim, heads=1) def forward(self, x, edge_index): x = F.elu(self.conv1(x, edge_index)) x = F.dropout(x, 0.6, training=self.training) return self.conv2(x, edge_index)
"GAT learns attention weights αᵢⱼ for each neighbor — so not all neighbors contribute equally. With 8 attention heads, the model captures 8 different relationship patterns in parallel, then concatenates them."

Key observations to mention

These are the 4 most important points to highlight verbally:

1. MLP ignores structure → lowest performance. Graph topology matters.
2. Handcrafted features hurt slightly → manual engineering adds noise; learned representations are better.
3. GCN vs GraphSAGE → same accuracy (72.1%) but GraphSAGE has better F1 and ROC-AUC, showing it generalizes better across classes.
4. GAT wins → attention mechanism selectively weights neighbors, which helps with CiteSeer's noisy, heterogeneous citation patterns.
