Model 4 — GraphSAGE

from torch_geometric.nn import SAGEConv class GraphSAGE(nn.Module): def __init__(self, in_dim, hidden_dim, out_dim): super().__init__() self.conv1 = SAGEConv(in_dim, hidden_dim) self.conv2 = SAGEConv(hidden_dim, out_dim) def forward(self, x, edge_index): x = F.relu(self.conv1(x, edge_index)) x = F.dropout(x, 0.5, training=self.training) return self.conv2(x, edge_index)
"Key difference from GCN: SAGEConv concatenates the node's own embedding with the aggregated neighbor embedding — it doesn't replace it. This makes GraphSAGE inductive: it can generalize to unseen nodes."

Model 5 — Graph Attention Network (GAT) ★ Best

from torch_geometric.nn import GATConv class GAT(nn.Module): def __init__(self, in_dim, hidden_dim, out_dim, heads=8): super().__init__() self.conv1 = GATConv(in_dim, hidden_dim, heads=heads, dropout=0.6) # 8 heads! self.conv2 = GATConv(hidden_dim * heads, out_dim, heads=1) # 1 head output def forward(self, x, edge_index): x = F.elu(self.conv1(x, edge_index)) # ELU activation x = F.dropout(x, 0.6, training=self.training) return self.conv2(x, edge_index)
"GAT learns attention weight αᵢⱼ for every neighbor — not all neighbors contribute equally. With 8 heads, it captures 8 different relationship patterns in parallel, then concatenates them."

Q&A — Dataset & Preprocessing

Expected Questions About Data

Why CiteSeer and not Cora or PubMed? ▼
"CiteSeer is a well-established benchmark with 6 classes. It's harder than Cora because of many isolated nodes — a better stress test for GNN robustness. PubMed is larger but simpler (3 classes). CiteSeer sits in the sweet spot for comparison."
What does the 3703-dim feature vector represent? ▼
"It's a bag-of-words vector — each dimension is a word in the vocabulary. Value = 1 means the word appears in that paper, 0 means it doesn't. It's binary and sparse — most of the 3703 values are 0."
Why 80:20 split instead of the standard 20-node-per-class split? ▼
"The standard benchmark uses only 120 training nodes (20 per class) — very small. I used 80:20 for a more realistic evaluation and fairer comparison across all 5 models. The tiny standard split is designed specifically for semi-supervised GNN evaluation."
Were there missing or noisy nodes in the dataset? ▼
"Yes — CiteSeer has nodes referenced in citation file but missing from content file. I filtered to keep only nodes that appear in both files, ensuring a clean node-feature mapping before building the graph."
What is edge_index and why shape [2, num_edges]? ▼
"edge_index is PyG's compact COO (coordinate) format. Row 0 = source nodes, Row 1 = destination nodes. So edge_index[:, i] = [src, dst] means paper src cites paper dst. Memory-efficient for sparse graphs."
Q&A — Models

Expected Questions About the Models

Key difference between GCN and GraphSAGE? ▼
"GCN is transductive — needs all nodes at training time, averages over ALL neighbors with symmetric normalization. GraphSAGE is inductive — samples a fixed neighborhood and CONCATENATEs the node's own embedding with aggregated neighbors. GraphSAGE can generalize to unseen nodes, GCN cannot."
How does GAT compute attention weights? ▼
"For each pair (i,j): concatenate [Whi ‖ Whj], pass through shared vector a with LeakyReLU → raw score. Softmax-normalize over all of node i's neighbors → weight αᵢⱼ. Final embedding = weighted sum of neighbor features using αᵢⱼ."
Why 8 heads in layer 1 but only 1 in layer 2? ▼
"8 heads in layer 1 lets the model capture 8 different relationship patterns simultaneously — each head learns a different attention distribution. The final layer needs 1 output per class, so we reduce to 1 head. This is the standard GAT architecture from Veličković et al."
Why did MLP + Graph Features perform worse than plain MLP? ▼
"Three reasons: (1) degree, PageRank, clustering are global scalar stats — tiny signal vs 3703 existing features; (2) CiteSeer's many isolated nodes make these features near-zero and noisy; (3) they may not align with class boundaries. End-to-end learned representations always win over manual engineering."
Why ReLU for GCN/SAGE but ELU for GAT? ▼
"ELU outputs smooth small negatives instead of hard zero for negative inputs. In GAT, attention-weighted features can have varied scales — ELU stabilizes gradients and training. The original GAT paper specifically used ELU."
Why is GAT dropout 0.6 but others use 0.5? ▼
"GAT has significantly more parameters due to 8 attention heads — more prone to overfitting on small CiteSeer. Higher dropout (0.6) acts as stronger regularization. This follows the recommendation in the original GAT paper."
Q&A — Results & Evaluation

Expected Questions About Results

Why is GCN's ROC-AUC (0.897) lower than plain MLP (0.910)? ▼
"GCN's symmetric normalization averages ALL neighbors uniformly. In CiteSeer's sparse, noisy graph, isolated or weakly connected nodes get poor aggregations — hurting class discrimination. MLP doesn't suffer from bad neighbor quality because it ignores the graph entirely. GraphSAGE avoids this by concatenating rather than replacing the node's own features."
Why macro F1 and not micro? ▼
"Macro F1 computes F1 per class and takes the unweighted average — treats all 6 classes equally regardless of size. CiteSeer has class imbalance, so micro F1 would be dominated by the majority class. Macro F1 gives a fairer view of minority class performance."
The accuracy differences look small — is this statistically significant? ▼
"The 4% gap between GAT and others is meaningful on a 6-class problem. However, I did not run multiple trials with different seeds — that's a limitation. GNN results on CiteSeer can vary 1–2% across seeds. Future work should report mean ± std over 5–10 runs."
How did you compute ROC-AUC for a 6-class problem? ▼
"Used sklearn.roc_auc_score with multi_class='ovr' (one-vs-rest) and average='macro'. For each class: compute AUC of that class vs all others, then take the macro average. Model outputs softmax probabilities for all 6 classes as the scores."
GCN and GraphSAGE have identical accuracy — coincidence? ▼
"Likely rounding to 4 decimal places. Also reflects a known phenomenon: on transductive benchmarks with fixed splits, many GNN architectures converge to similar accuracy because the dataset is the bottleneck. Differences show clearly in F1 and ROC-AUC."
Q&A — Theory & Concepts

Deep Theoretical Questions

What is the over-smoothing problem in GNNs? ▼
"With k layers you aggregate k-hop neighborhoods. With many layers, every node ends up averaging over almost the entire graph — all embeddings converge to the same vector, losing discriminative information. That's why I used only 2 layers. It's a fundamental limitation of current GNN architectures."
Transductive vs inductive learning? ▼
"Transductive (GCN): requires seeing all nodes including test nodes during training. Cannot generalize to new, unseen nodes. Inductive (GraphSAGE): learns a general aggregation function applicable to any node — including nodes not seen at training time. Critical for real applications where new papers or users appear over time."
What does D̂⁻½ Â D̂⁻½ normalization do exactly? ▼
"Â = A + I (adjacency with self-loops). D̂ is the diagonal degree matrix of Â. The symmetric normalization D̂⁻½ Â D̂⁻½ normalizes by both source AND target degree — prevents high-degree nodes from dominating aggregation."
Why do GNNs work well on citation networks specifically? ▼
"Homophily — papers that cite each other tend to be in the same research area. GNNs exploit this: by aggregating neighbor features, a node picks up signals from papers in the same topic cluster. If the graph had low homophily (random connections), GNNs would likely NOT outperform MLP."
What is message passing and how do all 3 GNN models use it? ▼
"Message passing: each node sends a message to neighbors, neighbors aggregate, node updates its representation. GCN: message = normalized feature, aggregate = sum. GraphSAGE: aggregate = mean, update = concat + linear. GAT: message = attention-weighted feature, aggregate = weighted sum. All three, but different aggregation strategies."
Why Adam and not SGD? ▼
"Adam adapts the learning rate per parameter using gradient moment estimates. For GNNs with sparse 3703-dim bag-of-words features, gradients vary wildly in magnitude. Adam handles this much better than SGD with a fixed learning rate, converging faster and more stably."
Q&A — Code & Implementation

Code-Level Questions

Why call model.train() and model.eval() separately? ▼
"model.train() activates dropout — randomly zeros neurons to prevent overfitting. model.eval() disables dropout so all neurons are active during evaluation — giving deterministic, stable predictions. Without switching, test accuracy would be artificially low."
What does training=self.training do in F.dropout()? ▼
"self.training is a boolean flag set by model.train() and model.eval(). Passing it makes dropout mode-aware automatically — active during training, disabled during evaluation. The correct way to use functional dropout in PyTorch."
Why CrossEntropyLoss and not NLLLoss? ▼
"CrossEntropyLoss = LogSoftmax + NLLLoss combined. The model outputs raw logits (no softmax applied), and CEL handles normalization internally. More numerically stable — avoids computing large softmax values explicitly."
How does train_mask work during loss computation? ▼
"Model outputs predictions for all 3312 nodes in one forward pass. train_mask is a boolean tensor [3312] — True for 80% of nodes. out[data.train_mask] selects only training nodes for loss. Gradient flows only through training nodes, but message passing still uses all edges."
Why call optimizer.zero_grad() every epoch? ▼
"PyTorch accumulates gradients by default — each backward() call adds to existing gradients. Without zero_grad(), gradients from previous epochs corrupt your weight updates. zero_grad() resets all parameter gradients to zero before computing new ones."
Could you have used Planetoid from PyG instead of manual loading? ▼
"Yes — Planetoid('CiteSeer') would do it automatically. I chose manual loading to understand the raw format and have full control over preprocessing — including handling missing nodes and creating the 80:20 split. It demonstrates deeper understanding of the data structure."
